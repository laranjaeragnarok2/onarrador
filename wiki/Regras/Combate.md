## Combate

O combate em Mighty Blade é um sistema dinâmico que determina o sucesso ou fracasso das ações dos personagens através de testes de dados. Ele abrange desde a iniciativa de cada rodada até as consequências de ataques e defesas.

### Ações e Testes

Sempre que um personagem desejar realizar uma ação, o Mestre determinará se um teste é necessário. Ações cotidianas e simples geralmente não requerem testes. Testes são necessários quando a complexidade da ação ou o estresse envolvido podem levar a um resultado incerto.

*   **Rolagem de Dados:** Para realizar um teste, o jogador rola uma quantidade de dados de seis lados (d6), geralmente 2d6, e soma o Atributo mais relevante para a ação.
*   **Dificuldade:** O resultado da rolagem é comparado com uma Dificuldade estabelecida pelo Mestre. Se o resultado for igual ou maior que a Dificuldade, o teste é bem-sucedido. Se for menor, o personagem falha.
*   **Inaptidão:** Se um personagem não possui a habilidade ou conhecimento necessário para uma ação, ou se as circunstâncias forem adversas (como lutar no escuro total), ele é considerado "Inapto" e rola apenas 1d6.
*   **Habilidades Adicionais:** Algumas Habilidades podem adicionar dados extras à rolagem de testes (+1d6, +2d6, etc.). Se múltiplas Habilidades concederem bônus de dados para a mesma situação, eles se acumulam.

#### Dados

O jogo utiliza exclusivamente dados de seis lados (d6). A notação "XdY" indica a quantidade de dados (X) a serem rolados, onde Y é o número de faces do dado (neste caso, sempre 6).

#### A Dificuldade dos Testes

A Dificuldade de um teste varia de acordo com a complexidade da tarefa. Tarefas com Dificuldade 7 ou menos são consideradas sucessos automáticos para agilizar o jogo. Testes são reservados para ações cujo sucesso ou fracasso impactam a narrativa.
