## Testes de Atributo

Os testes de atributo são a mecânica central de Mighty Blade para determinar o sucesso ou fracasso das ações de um personagem em situações que envolvem complexidade ou estresse. Quando um personagem tenta realizar algo que não é rotineiro, o Mestre pode exigir um teste para ver se a ação é bem-sucedida.

### Mecânica dos Testes

1.  **Anunciar a Intenção:** O jogador anuncia ao Mestre o que deseja que seu personagem faça.
2.  **Determinação do Teste:** O Mestre avalia se a ação requer um teste, considerando a complexidade e o estresse envolvido. Ações cotidianas geralmente não exigem testes.
3.  **Rolagem de Dados:** O jogador rola uma quantidade de dados de seis lados (d6) especificada (geralmente 2d6).
4.  **Soma do Atributo:** O resultado da rolagem dos dados é somado ao valor do Atributo mais relevante para a ação em questão.
5.  **Comparação com a Dificuldade:** O total obtido é comparado com um valor de Dificuldade estabelecido pelo Mestre.
6.  **Resultado:**
    *   Se o resultado da rolagem for **igual ou maior** que a Dificuldade, o teste é bem-sucedido.
    *   Se o resultado for **menor** que a Dificuldade, o personagem falha no teste.

### Dados Utilizados

Mighty Blade utiliza exclusivamente dados de seis lados (d6). A notação `XdY` indica a quantidade de dados de `Y` lados a serem rolados. Por exemplo, `2d6` significa rolar dois dados de seis lados.

### Quantos Dados Rolar

*   **Aptidão Padrão:** Em geral, quando um personagem tenta realizar uma tarefa, ele é considerado apto e rola **2d6**.
*   **Inaptidão:** Se a ação exigir uma habilidade ou conhecimento que o personagem não possui, ele é considerado **Inapto** para o teste e rola apenas **1d6**. Circunstâncias adversas (como lutar com alguém invisível ou na escuridão total) também podem levar à inaptidão. Se dois motivos para inaptidão se sobrepuserem, o personagem não poderá tentar a ação.
*   **Habilidades Adicionais:** Algumas Habilidades podem adicionar dados a testes específicos. Quando uma Habilidade indica "some +1d6 no teste", o jogador rola um dado a mais. Se múltiplas Habilidades conferirem bônus de +1d6 para a mesma situação, o personagem soma todos os bônus.

### A Dificuldade dos Testes

A Dificuldade de um teste é estabelecida pelo Mestre e reflete o quão simples ou complexa é a tarefa.

*   **Dificuldade 7 ou Menos:** Tarefas com Dificuldade 7 ou menos são consideradas sucessos automáticos para agilizar o jogo. Testes não são necessários para atividades rotineiras.
*   **Ações Significativas:** Testes são necessários para ações cujo fracasso ou sucesso podem mudar o curso da história.
*   **Tempo e Risco:** Se um personagem tem tempo de sobra e não corre um risco considerável, a tarefa pode ser considerada automaticamente bem-sucedida pelo Mestre, que estipulará um tempo razoável para a conclusão.
